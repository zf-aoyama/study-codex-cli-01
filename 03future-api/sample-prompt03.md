目的
無料かつ認証不要で利用できる外部APIからBTCの円建て価格を取得し、
BTCマイニングの簡易収益計算を行うCLIツールをPythonで作成してください。

進め方
- まず要件を満たす実装方針を自分で設計してください
- その上で、必要なファイル構成とコードを作成してください
- 実装前に、採用する価格取得APIが無料かつ認証不要であることを公式ドキュメントで確認してください
- APIの具体仕様は自分で調査し、正しい実装に反映してください
- 技術記事や個人ブログより、公式ドキュメントを優先してください

開発条件
- Pythonプロジェクトは uv で管理する
- `pyproject.toml` を作成する
- `uv sync` でセットアップできること
- `uv run python main.py` で実行できること
- 標準ライブラリ中心で実装する
- 不要な追加ライブラリは使わない
- requirements.txt は作成しない

ツール仕様
CLIで以下を入力できること
- ハッシュレート (TH/s)
- 消費電力 (W)
- 電気代 (円/kWh)
- 想定採掘量 (BTC/日)

外部APIから取得すること
- BTC価格 (円/BTC)

出力すること
- 取得したBTC価格（円/BTC）
- 1日の電気代（円）
- 1日の売上（円）
- 1日の利益（円）
- 30日間の利益（円）

計算式
- 1日の電気代 = (消費電力 / 1000) * 24 * 電気代
- 1日の売上 = 想定採掘量 * BTC価格
- 1日の利益 = 1日の売上 - 1日の電気代
- 30日間の利益 = 1日の利益 * 30

実装要件
- CLIでユーザー入力を受け取ること
- 入力値は float として扱うこと
- 0以下の値が入力された場合は日本語でエラーメッセージを表示すること
- 数値以外が入力された場合も日本語でエラーメッセージを表示すること
- API通信失敗時は日本語で分かりやすくエラーメッセージを表示すること
- HTTPエラー、タイムアウト、JSON解析エラー、想定外レスポンスを適切に処理すること
- 出力は日本語で見やすく表示すること
- 金額は桁区切りで表示すること
- コードは読みやすく保守しやすくすること
- 過剰な抽象化は避けること
- ただし将来的なAPI差し替えはしやすい構成にすること
