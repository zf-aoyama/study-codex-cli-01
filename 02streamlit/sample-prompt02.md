目的
BTCマイニングの簡易収益を計算するStreamlit製GUIツールをPythonで作成する。

開発条件
- Pythonプロジェクトは uv で管理する
- `pyproject.toml` を作成する
- `uv sync` で依存関係をインストールできること
- `uv run streamlit run app.py` で起動できること
- 使用ライブラリは必要最小限にする
- Streamlit以外の追加ライブラリは基本的に使わない

作成するファイル
- app.py
- pyproject.toml
- README.md

ツール仕様
1ページ構成のシンプルなGUIツールを作る。

入力項目
- ハッシュレート (TH/s)
- 消費電力 (W)
- 電気代 (円/kWh)
- BTC価格 (円/BTC)
- 想定採掘量 (BTC/日)

出力項目
- 1日の電気代（円）
- 1日の売上（円）
- 1日の利益（円）
- 30日間の利益（円）

計算式
- 1日の電気代 = (消費電力 / 1000) * 24 * 電気代
- 1日の売上 = 想定採掘量 * BTC価格
- 1日の利益 = 1日の売上 - 1日の電気代
- 30日間の利益 = 1日の利益 * 30

実装要件
- 入力は数値入力フォームで受け取ること
- 小数入力に対応すること
- 0以下の値は受け付けず、エラーメッセージを表示すること
- 計算処理は関数に分離すること
- 表示は日本語にすること
- 金額は桁区切りで見やすく表示すること
- 画面はシンプルで見やすくすること
- 余計な機能は追加しないこと

UI要件
- タイトルを表示すること
- 入力エリアを上部にまとめること
- 計算ボタンを1つ配置すること
- 計算結果を下部に見やすく表示すること
- 結果は4項目をそれぞれ分かりやすく表示すること
- 可能なら `st.metric` を使って見やすく表示すること
- 配色やデザインはシンプルにすること
- グラフは不要
- 複数ページ化しないこと
- サイドバーは不要

関数要件
以下の関数を作ること
- calculate_daily_electric_cost
- calculate_daily_revenue
- calculate_daily_profit
- calculate_monthly_profit

app.pyの構成要件
- 上記の計算関数を定義する
- `main()` 関数を作成する
- `if __name__ == "__main__":` で `main()` を呼び出す構成にする
